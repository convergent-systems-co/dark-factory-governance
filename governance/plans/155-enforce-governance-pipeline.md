# Enforce Dark Governance Framework Across All Modes

**Author:** Code Manager (agentic)
**Date:** 2026-02-23
**Status:** completed
**Issue:** https://github.com/SET-Apps/ai-submodule/issues/155
**Branch:** `itsfwcp/fix/155/enforce-governance-pipeline`

---

## 1. Objective

Ensure the Dark Factory governance pipeline is always enforced regardless of operating mode (local or remote). Missing panels must cause explicit failure with remediation guidance. Default panels from the AI-submodule must always run. GOALS.md must be created during `init.sh` for consuming repos.

## 2. Rationale

The issue reports that in local-mode, the AI agent bypassed the governance pipeline (no plan created, direct implementation). This happened because:
- The governance enforcement rules in `instructions.md` are too brief and don't mention local-mode enforcement
- There's no explicit "governance gate" in the instructions that makes the plan-first workflow mandatory
- The `dark-factory-governance.yml` workflow's `skip-review` job only warns (does not block) when no emissions exist
- `init.sh` does not create a GOALS.md for consuming repos

| Alternative | Considered | Rejected Because |
|-------------|-----------|------------------|
| Runtime enforcement daemon | Yes | Out of scope for config-only repo; blocked by tool limitations |
| Pre-commit hook enforcement | Yes | Only catches at commit time, not at planning stage; adds complexity |
| Instruction-level enforcement + CI hardening | Yes (selected) | Best approach — enforces at the cognitive layer (instructions) and CI layer (workflow) |

## 4. Approach

1. **Strengthen `instructions.md`** — Add a "Governance Pipeline" section that makes plan-first mandatory and states the governance workflow must execute regardless of mode (local/remote). This is the cognitive-layer enforcement.

2. **Expand `instructions/governance.md`** — Add:
   - Local-mode enforcement rules (same pipeline, just without GitHub-specific steps)
   - Required default panels list (from `default.yaml`)
   - Panel validation: if emissions directory is empty or required panels missing, flag as governance failure
   - Remediation guidance for missing panels

3. **Harden `dark-factory-governance.yml`** — Change the `skip-review` job to:
   - Check for required panels from `default.yaml`
   - Post a request-changes review (not just a comment) when required panels are missing
   - Exit with failure so the PR check blocks merge
   - Include specific remediation steps in the review body

4. **Update `init.sh`** — Add:
   - GOALS.md generation from template (only in submodule context, only if not already present)
   - Panel emission validation: check that required panels have emission files

5. **Create `templates/GOALS.md`** — Template based on the ai-submodule's own GOALS.md structure but generalized for consuming repos.

6. **Update documentation** — GOALS.md, CLAUDE.md as needed.

## 8. Backward Compatibility

All changes are additive. The `skip-review` job change is intentionally stricter (from warning to blocking), which is the desired behavior per the issue. Consuming repos that intentionally have no panels can set `governance.skip_panel_validation: true` in their `project.yaml` to opt out.
